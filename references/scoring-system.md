# 评分系统

## 维度权重

### 完整模式（有 API Key）

| 维度 | 检查项数 | 权重 | 理由 |
|------|----------|------|------|
| 技术 SEO | 25 项 | 30% | 基础门槛 + Core Web Vitals |
| 页面元素 | 20 项 | 20% | 直接影响 SERP 展示和点击率 |
| 内容质量与 E-E-A-T | 28 项 | 50% | Google 核心排名信号 |
| **总计** | **73 项** | **100%** | |

### 降级模式（无 API Key）

当用户未配置 PageSpeed API Key 时，Core Web Vitals（8 项）无法检查，自动切换到降级模式：

| 维度 | 检查项数 | 权重 | 理由 |
|------|----------|------|------|
| 技术 SEO | 17 项（跳过 T2.1-T2.8） | 25% | 基础技术检查 |
| 页面元素 | 20 项 | 20% | 直接影响 SERP 展示和点击率 |
| 内容质量与 E-E-A-T | 28 项 | 55% | Google 核心排名信号 |
| **总计** | **65 项** | **100%** | |

> **说明**: 无 API Key 时，E-E-A-T 权重提升，因为这是 Google 最重要的排名信号。

## E-E-A-T 子维度权重

| 子维度 | 检查项数 | 占 E-E-A-T 权重 | 占总权重 |
|--------|----------|-----------------|----------|
| Trust（信任度） | 8 项 | 40% | 20% |
| Experience（经验） | 7 项 | 20% | 10% |
| Expertise（专业度） | 7 项 | 20% | 10% |
| Authoritativeness（权威性） | 6 项 | 20% | 10% |

> Trust 权重最高，因为 Google 明确表示"Trust is the most important member of the E-E-A-T family"。

## 单项评分标准

每个检查项采用三级评分：

| 状态 | 符号 | 得分 | 含义 |
|------|------|------|------|
| 优秀 | 🟢 | 100 | 完全符合标准 |
| 需改进 | 🟡 | 50 | 部分符合或有改进空间 |
| 严重问题 | 🔴 | 0 | 不符合标准或缺失 |

## 维度得分计算

### 方法 1：简单平均（推荐）

```
维度得分 = (Σ 各检查项得分) / 检查项数量
```

**示例：技术 SEO 维度**
- 25 个检查项
- 20 个 🟢（100分）+ 3 个 🟡（50分）+ 2 个 🔴（0分）
- 维度得分 = (20×100 + 3×50 + 2×0) / 25 = 86 分

### 方法 2：加权平均（可选）

对于特别重要的检查项（如 LCP、H1），可设置更高权重。

## 综合得分计算

```
综合得分 = 技术SEO得分 × 30% + 页面元素得分 × 20% + E-E-A-T得分 × 50%
```

**示例：**
- 技术 SEO：86 分
- 页面元素：78 分
- E-E-A-T：72 分
- 综合得分 = 86×0.3 + 78×0.2 + 72×0.5 = 25.8 + 15.6 + 36 = **77.4 分**

## 等级划分

| 等级 | 分数范围 | 符号 | 说明 |
|------|----------|------|------|
| A | 90-100 | 🟢 | 优秀，保持当前水平 |
| B | 80-89 | 🟢 | 良好，小幅优化 |

| D | 60-69 | 🟡 | 较差，重点改进 |
| F | <60 | 🔴 | 不及格，急需修复 |

## 优先级划分

### P0：必须立即修复

严重影响索引或用户体验的问题：

- robots.txt 阻止重要页面
- noindex 标签错误使用
- HTTPS 未启用
- 页面返回 4xx/5xx 错误
- H1 缺失
- 严重的 CLS (>0.25)

### P1：尽快修复

影响排名或用户体验的问题：

- Title/Meta Description 缺失或过长
- LCP >2.5s
- 图片无 Alt 属性
- 无 canonical 标签
- 内部链接过少

### P2：计划优化

改进空间，非紧急：

- Open Graph 标签缺失
- Twitter Card 缺失
- 结构化数据不完整
- 作者信息不详尽

## 快速评分卡

用于快速手工评估（满分 100 分）：

| 维度 | 权重 | 评分标准 (0-10) |
|------|------|-----------------|
| 技术基础 | 15% | 错误多(0) → 部分通过(5) → 全部通过(10) |
| 页面元素 | 15% | 缺失严重(0) → 基本完整(5) → 优化完善(10) |
| 作者实体 | 15% | 无署名(0) → 有署名但无背景(5) → 权威专家+Schema(10) |
| 网站品牌 | 15% | 三无网站(0) → 信息完整(5) → 行业知名品牌(10) |
| 内容深度 | 20% | 纯洗稿(0) → 整合信息(5) → 独家数据/极高努力值(10) |
| 引用与准确 | 10% | 无引用/有错(0) → 普通引用(5) → 引用权威源(10) |
| 用户体验 | 10% | 广告遮挡/慢(0) → 正常(5) → 极速/排版优美(10) |

**目标分数：≥ 80 分**
