## 1. AI引用格式优化

### A1 TL;DR摘要存在性

| 属性 | 值 |
|------|-----|
| 执行方式 | AI 分析页面开头是否有摘要 |
| 对应信号 | AI 引用友好度 |
| 评分标准 | 有 TL;DR/摘要🟢 / 无🟡 |

**原理**: AI搜索引擎偏好引用包含简洁摘要的页面,因为摘要能直接回答用户问题。

**检查要点:**
- 页面开头是否有"摘要"、"TL;DR"、"要点"等标识
- 摘要是否在3-5句话内概括核心内容
- 摘要是否使用要点列表形式

**最佳实践:**
```html
<div class="summary">
  <h2>TL;DR</h2>
  <ul>
    <li>核心要点 1</li>
    <li>核心要点 2</li>
    <li>核心要点 3</li>
  </ul>
</div>
```

## 2. Answer-Oriented Writing

### A2 先结论后支撑的写作模式

| 属性 | 值 |
|------|-----|
| 执行方式 | AI 分析每个section的结构 |
| 对应信号 | AI 直接引用适配度 |
| 评分标准 | 多数section采用先结论后支撑🟢 / 部分采用🟡 / 不采用🔴 |

**原理**: AI偏好引用直接给出答案的内容块,而非需要阅读全文才能找到结论的内容。

**检查要点:**
- 每个H2/H3下第一段是否直接给出答案
- 是否避免"故事化开头"直接进入主题
- 关键信息是否前置

**好的模式:**
```
## 如何提高页面加载速度?

**答案**: 优化图片大小、启用浏览器缓存、使用CDN。

**详细说明**: [展开具体步骤]
```

**差的模式:**
```
## 页面加载速度的重要性

在现代互联网时代,用户期望... [冗长铺垫]
因此,优化速度非常重要... [仍未给出方法]
```

## 3. 可引用内容块

### A3 独立内容单元数量

| 属性 | 值 |
|------|-----|
| 执行方式 | AI 识别表格、代码块、步骤列表、定义框 |
| 对应信号 | 内容模块化程度 |
| 评分标准 | ≥5个独立块🟢 / 2-4个🟡 / \u003c2个🔴 |

**原理**: AI搜索引擎更容易摘录格式清晰、能独立理解的内容块(如表格、代码示例)。

**可引用的内容块类型:**
- 对比表格 (功能对比、价格对比)
- 步骤列表 (操作指南)
- 代码示例
- 定义框/提示框
- 统计数据表格

**示例:**
```html
<table>
  <caption>SEO工具对比</caption>
  <tr>
    <th>工具</th>
    <th>价格</th>
    <th>特点</th>
  </tr>
  <!-- ... -->
</table>
```

## 4. FAQ Schema

### A4 FAQPage结构化数据

| 属性 | 值 |
|------|-----|
| 执行方式 | 解析 JSON-LD 中的 `@type: FAQPage` |
| 对应信号 | AI 问答展示 |
| 评分标准 | 有FAQ Schema🟢 / 有FAQ但无Schema🟡 / 无FAQ🔴 |

**原理**: FAQPage结构化数据让AI搜索引擎直接提取问答对,提高在AI回答中被引用的概率。

**正确实现:**
```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "什么是SEO?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "SEO(搜索引擎优化)是提高网站在搜索结果中排名的技术。"
    }
  }]
}
```

## 5. 语音搜索优化

### A5 自然语言问答格式

| 属性 | 值 |
|------|-----|
| 执行方式 | AI 分析内容是否包含完整问句和句子形式回答 |
| 对应信号 | 语音助手引用适配度 |
| 评分标准 | 包含问答对且回答是完整句子🟢 / 仅关键词🟡 / 无🔴 |

**原理**: 语音助手倾向朗读完整句子,而非关键词列表,因此内容应使用自然语言。

**检查要点:**
- 是否包含"什么是"、"如何做"类问句
- 回答是否是可朗读的完整句子
- 避免仅用关键词或表格回答

**好的示例:**
```
Q: SEO审计需要多久?
A: 一次完整的SEO审计通常需要2-4小时,具体取决于网站规模。
```

**差的示例:**
```
审计时长: 2-4小时
```
